# 📋 PRD 审查 Checklist 模板（专业可执行版）

> 使用方式：
>
> * 每项 ✔ / ❌ / ⚠
> * 关键项必须全 ✔ 才能进入开发
> * 建议结合评分制（后附）

---

# 一、目标与背景（Why 层）

## 1. 业务目标是否明确？

* [ ] 是否明确说明本期解决什么问题？
* [ ] 是否定义成功指标（可量化）？
* [ ] 是否与公司/项目阶段目标一致？
* [ ] 是否说明优先级来源？

**否决条件：**

> 如果无法回答“为什么现在做这个？” → 不进入开发。

---

# 二、用户与场景（Who / When）

## 2. 用户定义是否清晰？

* [ ] 是否明确目标用户是谁？
* [ ] 是否区分主用户与边缘用户？
* [ ] 是否定义使用场景？
* [ ] 是否说明频率与关键路径？

**风险信号：**

> “所有用户都需要” → 通常不够聚焦。

---

# 三、范围定义（Scope Control）

## 3. 是否定义功能范围？

* [ ] 是否列出本期功能清单？
* [ ] 是否明确 Out of Scope？
* [ ] 是否避免隐性功能（未写但默认存在）？
* [ ] 是否有版本拆分说明（如 v1/v2）？

**否决条件：**

> 如果边界不清晰 → 必然 scope creep。

---

# 四、功能规格（What）

## 4. 功能是否可直接实现？

* [ ] 每个功能是否描述完整？
* [ ] 是否有交互流程说明？
* [ ] 是否明确字段定义？
* [ ] 是否描述所有按钮行为？
* [ ] 是否定义状态变化逻辑？
* [ ] 是否描述排序规则 / 计算规则？
* [ ] 是否明确权限控制逻辑？

**核心判断：**

> 研发是否无需口头补充即可开发？

---

# 五、异常与边界（Edge Cases）

## 5. 是否覆盖异常情况？

* [ ] 网络异常如何处理？
* [ ] 空数据如何展示？
* [ ] 权限不足如何反馈？
* [ ] 接口超时如何处理？
* [ ] 并发冲突如何处理？
* [ ] 数据异常如何兜底？

**否决条件：**

> 如果只写理想路径 → 不合格。

---

# 六、非功能需求（NFR）

## 6. 是否定义性能与约束？

* [ ] 是否定义性能要求？
* [ ] 是否定义兼容性要求？
* [ ] 是否定义安全要求？
* [ ] 是否定义数据规模预期？
* [ ] 是否定义可扩展性约束？

---

# 七、可测试性（Testability）

## 7. 是否可写测试用例？

* [ ] 是否定义验收标准？
* [ ] 是否定义完成标准？
* [ ] 是否定义数据验证方式？
* [ ] 是否定义回归影响范围？

**核心判断：**

> QA 是否可以仅根据 PRD 写测试用例？

---

# 八、逻辑一致性（Consistency）

## 8. 是否前后一致？

* [ ] 是否存在逻辑冲突？
* [ ] 是否存在目标与设计不匹配？
* [ ] 是否存在自相矛盾？
* [ ] 是否与历史版本冲突？

---

# 九、依赖与影响分析（Impact）

## 9. 是否识别依赖？

* [ ] 是否明确依赖系统？
* [ ] 是否明确接口依赖？
* [ ] 是否评估影响模块？
* [ ] 是否评估数据迁移？
* [ ] 是否识别上线风险？

---

# 十、决策透明度（Decision Record）

## 10. 是否记录关键决策？

* [ ] 是否说明方案选择原因？
* [ ] 是否记录被否决方案？
* [ ] 是否有数据支持？

---

# 🚦PRD 通过标准

建议三层标准：

### 🟢 可直接开发

* 无 ❌
* 关键项全 ✔

### 🟡 可开发但有风险

* ≤ 3 个 ⚠
* 无关键缺失

### 🔴 不允许进入开发

* 任意关键项 ❌
* 目标不清
* 边界未定义
* 无验收标准

---

# 📊 可选：量化评分模型（会议用）

| 模块    | 分值 |
| ----- | -- |
| 目标清晰  | 10 |
| 范围清晰  | 15 |
| 功能完整  | 20 |
| 边界覆盖  | 15 |
| 可测试性  | 15 |
| 非功能需求 | 10 |
| 依赖分析  | 10 |
| 一致性   | 5  |

85 分以上 → 高质量
70–85 → 可优化
70 以下 → 重写

---

# 🧠 高级判断法（高手用）

问三个问题：

1. 如果产品经理离职，这个 PRD 是否还能被正确实现？
2. 如果半年后复盘，是否能看懂当时的决策逻辑？
3. 如果发生线上事故，是否可以从 PRD 追溯原因？

如果答案是否 → PRD 不成熟。
