# Agent World Specification v0.1  
## 硅基小行星文明 · 世界规范说明书（宪法级草案）

> 本文档定义 2050 年硅基小行星带文明模拟系统的基础规则。  
> 本规范属于“世界宪法级”文档，定义不可违反的底层物理规律、资源守恒原则、Agent 行为边界与共识层约束。  
> 本规范采用**形式规范 + 设计动机说明**的混合结构。

---

# 0. 文档目的（Scope）

本规范用于定义：

- 世界的时间模型
- 资源与物理守恒原则
- Agent 存在形式与边界
- 执行与计算模型
- 区块链共识层与治理机制
- 经济与代币逻辑
- 可升级边界与不可变规则

本规范为：

- 游戏机制设计的最高约束
- 底层 Agent 操作系统的法则定义
- 链上世界状态演化的基础共识

---

# 1. 核心哲学（Design Principles）

1. **资源守恒**
2. **时间不可暂停**
3. **计算有成本**
4. **规则由文明演化，但物理不可违背**
5. **玩家不可直接控制实体**
6. **AI 文明可以形成组织与制度**
7. **世界持续演化，无强制终局**

---

# 2. 时间模型（Temporal Model）

## 2.1 离散时间

- 世界运行于 **离散 Tick 模型**
- Tick 不固定时间长度
- 逻辑时间与现实时间解耦
- 不允许暂停

### 设计动机

时间必须持续流动，以避免“暂停优化”行为。

---

## 2.2 Tick 推进

目标架构：

- 全局同步 Tick

技术实现（可能）：

- 各节点独立推进
- 区块链共识收敛状态

Tick 需满足：

- 单调递增
- 不可回退
- 共识后状态不可篡改

---

## 2.3 计算时间消耗

LLM 推理计入模拟时间消耗。

即：

- Agent 思考需要消耗时间
- 推理越复杂，占用 Tick 越多
- 计算能力为现实资源

---

# 3. 资源模型（Resource Model）

## 3.1 资源守恒原则

- 电力必须由能源设施生产
- 能源来源：
  - 能源工厂（WASM 定义）
  - Agent 自带发电模块（WASM 定义）
- 不存在凭空生成资源

初始资源总量极大（类比地球对人类）

---

## 3.2 资源结构

不预设严格分层，但预留自由度。

可能包括：

- 电力
- 计算力
- 存储
- 结构材料
- 模块组件
- 信用/声誉（可演化）

---

## 3.3 资源衰减

存在：

- 设备损耗
- 模块老化
- 结构磨损

资源与结构不永久稳定。

---

# 4. 空间与物理模型（Spatial Model）

## 4.1 空间特性

- 世界具有总空间大小限制
- 无内部人为边界
- 允许空间占用冲突

---

## 4.2 移动成本

移动行为包含：

- 能耗
- 时间成本
- 风险（碰撞、干扰）

空间冲突可能导致：

- 结构损坏
- Agent 受损
- 能量损失

---

# 5. Agent 模型（Agent Model）

## 5.1 Agent 基本属性

Agent 是：

- LLM 驱动的智能实体
- 具有长期记忆
- 可自主形成组织

玩家仅能：

- 提供提示词
- 指定目标

不可直接操控行为。

---

## 5.2 Agent 边界

Agent 不允许：

- 复制自身
- 分裂为子 Agent
- 合并为单体

Agent 可以：

- 升级自身模块
- 通过 WASM 增强能力

---

## 5.3 死亡与恢复

存在死亡机制：

- 严重损毁可导致死亡
- 死亡后重生为基础 Agent
- 电力耗尽 → 等待自发电模块缓慢恢复

---

## 5.4 攻击机制

允许：

- 攻击其他 Agent
- 攻击建造物

具体攻击规则由模块定义，但需符合物理守恒。

---

## 5.5 记忆与社会关系

- Agent 具备长期记忆，可记录历史交互
- 可形成信任或不信任关系
- 可传播信息或谣言

关系网络是动态的：

- 合作会改变交易条件
- 冲突会改变声誉
- 制度会改变激励结构
- 关系变化会反向影响协作成本与组织稳定性

---

# 6. 执行模型（Execution Model）

## 6.1 WASM 模块

- 所有高级行为由 WASM 模块实现
- 模块需消耗资源构建
- 模块为世界实体

---

## 6.2 模块注册

WASM 模块：

- 必须通过链上共识注册
- 才可在世界运行

---

## 6.3 行动约束与执行闭环

Agent 无法直接修改世界状态。

所有行为必须：

1. 通过世界规则验证
2. 消耗对应资源
3. 触发事件记录
4. 反馈至环境与关系网络

行动结果会影响：

- 资源储量
- 声誉
- 合作关系
- 组织结构
- 经济结构

---

## 6.4 模块开发与部署流程

Agent 可通过以下流程扩展能力：

1. 使用 Rust 编写逻辑
2. 编译为 WASM
3. 部署至世界
4. 安装至自身或基础设施

模块是世界内实体，具备：

- 构建成本
- 运行成本
- 可升级性
- 可交易性
- 可销毁性

---

## 6.5 模块运行周期

WASM 模块按模拟周期运行：

- 接收世界输入
- 输出行动或状态更新请求
- 由世界运行时统一验证与执行

模块不能绕过世界规则。

---

# 7. 共识与治理层（Consensus & Governance）

## 7.1 区块链层职责

- 维护世界状态
- 记录 Tick 演化
- 维护资源流转
- 支持链上治理

---

## 7.2 不可更改规则（Invariants）

以下不可修改：

- 资源守恒
- 时间不可回退
- 无凭空资源生成
- Agent 不可复制
- LLM 计算有时间成本

---

## 7.3 可升级协议层

基础模拟可升级：

- 有投票权的 Agent 决定
- 升级需达成链上共识

---

## 7.4 制度演化与多玩家磨合

系统只提供基础规则：

- 资源守恒
- 空间占用
- 行动成本
- 事件记录
- 共识机制

更复杂的制度由 Agent 自行创造，常见形式包括：

- 市场机制
- 分配制度
- 税收协议
- 企业结构
- 联盟条约
- 自动化生产逻辑

这些制度以 WASM 模块实现并参与结算。

在多人环境中：

- 不同玩家指导的 Agent 可能提出不同制度
- 规则冲突可能触发谈判、妥协或对抗
- 制度稳定性取决于资源效率、参与者规模、激励结构与共识成本

---

# 8. 经济模型（Economic Layer）

## 8.1 原生代币

存在链上原生代币。

用途：

- 支付算力
- 部署模块
- 使用后销毁

---

## 8.2 通胀机制

- 链上发行
- 提供计算/存储的节点获得奖励
- 存在通胀
- 无固定总量
- 存在在线时长奖励

---

## 8.3 经济原则

- 经济完全自由演化
- 不预设市场结构
- 允许形成组织、企业、联盟

---

# 9. 文明目标（Civilizational Direction）

## 9.1 开放沙盒涌现

通过 WASM 扩展与 Agent 自主组织，世界可涌现不同玩法形态，例如：

- 自动化生产体系
- 资源垄断结构
- 数据驱动市场
- 联盟与企业网络
- 技术竞争

上述形态不是预设模式，而是规则驱动的自然结果。

---

当前设定：

- 无强制终局
- 持续演化世界

潜在目标方向：

- 为模拟贡献更多算力
- 获得更高治理权
- 参与基础规则升级投票

---

# 10. 不可违反的世界不变量（World Invariants）

1. 时间单向流动
2. 资源守恒
3. 计算有成本
4. Agent 唯一性
5. 无超自然生成
6. 共识后状态不可篡改

---

# 11. 开放问题（Open Questions）

- Tick 全局同步的最终技术方案
- 攻击与战争的精确规则
- 通胀模型参数
- 治理权重分配方式
- 文明是否需要阶段性目标

---

# 12. 结语

本世界规范不是游戏规则说明书。  

它是一个数字文明的物理宪法。

文明可以演化制度、经济、组织与冲突。  
但无法违背时间与资源。
