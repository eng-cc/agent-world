为什么是 agent-world

从“会说话”到“可治理”

Agent 行为不是黑箱对话，而是必须通过规则约束与状态校验，结果可追责、可解释。

从“一次运行”到“持久世界”

快照与事件日志让仿真实验可恢复、可回放、可比较，形成稳定的研究与工程基线。

从“单智能体”到“文明涌现”

多主体在资源、制度、关系中长期演化，支持观察协作、竞争、冲突与秩序形成过程。

演示与快速开始

离线回放（推荐首看）

生成世界数据后启动 viewer server + UI，适合复盘与分析。

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

在线演化（可选 LLM）

live server 持续推进世界状态，viewer 实时订阅演化过程。

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- twin_region_bootstrap --bind 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

3 步上手

  1. 克隆仓库并进入项目目录。
  2. 执行 `env -u RUSTC_WRAPPER cargo test` 验证环境。
  3. 运行 demo / live 命令观察世界演化与回放。

若需 LLM 驱动，复制 `config.example.toml` 为 `config.toml`，并配置 `AGENT_WORLD_LLM_MODEL/BASE_URL/API_KEY`。

FAQ

这个项目和普通 Agent Demo 最大区别是什么？

核心差异是“世界约束 + 持久回放 + 可审计证据链”，关注长期演化而非短时对话表现。

不接入 LLM 可以运行吗？

可以。系统支持无 LLM 的规则驱动模式，可先验证世界机制再逐步接入模型。

最低上手路径是什么？

执行 demo 生成世界数据，再启动 viewer server 与 viewer 客户端即可看到完整演化轨迹。

为什么强调 replay 与 audit？

因为“可复现 + 可解释”是多主体系统进入工程化与研究化的前提。

是否支持自定义模块？

支持，当前采用 Rust + WASM 的模块扩展策略，便于逐步接入新规则与新能力。

怎么快速了解项目进展？

优先看 README、设计文档与开发日志，路线图和里程碑会持续更新。
