oasis7 world shape
Slide 02

我更想做一个
会自己持续运转的世界

不是玩家上线时世界才开始动，也不是所有变化都靠人工触发。这个世界应该在规则之下自己推进。

World Signal
persistent simulation
它先是一个系统，再是一组内容。

先让世界成立，再让里面的关系、组织和冲突慢慢出现。

Why I Care
我想看的不是“通关路线”。

而是一个世界在没有预写答案的情况下，能不能长出自己的秩序感。

What Changes
  • 故事不再完全靠手写推进。
  • 系统关系会比任务清单更重要。
  • 玩家看到的是世界的演化，不只是关卡顺序。
world shape 2 / 7
oasis7 rule stack
Slide 03

给它资源约束、空间约束、交易规则和协作压力

真正有意思的，不是“它会不会表现得像人”，而是当规则开始互相顶住时，会不会出现稳定的分工和真实的冲突。

Constraint Model
pressure drives behavior
秩序不是讲出来的。

它要在资源有限、位置有限、协作有成本的情况下，自己变得有必要。

Pressure Sources
  • 能源和数据不是无限的。
  • 空间占用和移动都要付代价。
  • 交易和合作不是默认成功。
What I Want To See
当约束叠起来，秩序才会开始长。

分工、组织和冲突都应该是被系统逼出来的，而不是被文案塞进去的。

Slide 04

背景会更接近一片
小行星带

里面不是传统的人类城镇，而是一群硅基智能体，靠能源、算力、数据这些东西活下去。

Setting Signal
hard sci-fi habitat
这个背景不是为了“酷”。

它本身就更适合把资源稀缺、空间关系和制度演化讲清楚。

Why Asteroids
因为这里天然有距离、轨道、基础设施和争夺。

很多我想要的压力，在这个环境里会显得更自然，也更系统化。

Life Support
  • 能源决定持续性。
  • 算力决定行动能力。
  • 数据决定记忆与协作质量。
asteroid belt setting 4 / 7
oasis7 player position
Slide 05

玩家不是直接操控角色
更像是在世界外面给方向的人

你可以影响它，但不能直接替它行动。玩家的乐趣不是“附身控制”，而是长期引导和结构塑形。

Player Model
indirect control
我不想把它做成木偶戏。

玩家不是按按钮替世界做决定，而是在边界里慢慢改变它的发展方向。

Player Can
  • 给目标和长期方向。
  • 调整策略和发展路线。
  • 帮助世界扩展能力。
Player Cannot
  • 直接替某个角色按下一步动作。
  • 跳过世界规则改结果。
  • 把复杂系统拍扁成单角色操作。
player as guide 5 / 7
oasis7 emergent change
Slide 06

我想看的不是固定答案
而是一个世界怎么慢慢长出分工、组织和冲突

我最想要的，不是漂亮设定先行，而是系统真的把这些变化逼出来，然后被人看见、介入、影响。

End State
order emerges
真正迷人的地方在“长出来”。

如果一切都预写好了，世界就只是被表演；只有在规则里长出来，它才真的像活的。

What Emerges
  • 分工不是职位表，而是压力下的选择。
  • 组织不是设定词，而是协作成本的结果。
  • 冲突不是剧情按钮，而是利益和边界的碰撞。
Why This Project
所以前面那支 agent 队伍不是噱头。

因为我要做的，本来就是一个更适合用系统和协作一起推进的游戏。
