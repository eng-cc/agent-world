Core Capabilities

Four capability blocks only, so you can quickly decide if this stack matches your research or product goals.

Rule Execution Kernel

Every action goes through policy checks, with denials and consequences recorded.

Value: governance and explainability by default

Persistent Replay

Snapshots and journals enable deterministic restore, compare, and postmortem analysis.

Value: reproducible and auditable experiments

Multi-agent World Modeling

Agents evolve under resource, relation, and institutional constraints over long horizons.

Value: observe cooperation, conflict, and order emergence

LLM / WASM Extension Layer

OpenAI-style model integration plus modular rule extensions for iterative complexity growth.

Value: stronger policies without destabilizing the core loop

Architecture Loop

Runtime

Unified action gateway, policy validation, and execution boundaries.

Simulator

Advances world time and state while writing snapshots and event logs.

Viewer

Supports both offline replay and online subscription for observability.

Demo and Quick Start

Offline Replay (Recommended)

Generate world data, start replay server, then inspect everything in the viewer.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Live Evolution (Optional LLM)

Run the live server and subscribe from the viewer to observe ongoing state transitions.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- twin_region_bootstrap --bind 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Three-step First Run

  1. Clone the repository and enter the workspace.
  2. Run `env -u RUSTC_WRAPPER cargo test` to validate the environment.
  3. Start demo or live flow and inspect events in viewer.
