Demo & Quick Start

Offline Replay (recommended first)

Generate world data then launch viewer server + UI for replay and analysis.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Online Evolution (optional LLM)

Run live server continuously while viewer subscribes to state updates in real time.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- twin_region_bootstrap --bind 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

3-step start

  1. Clone this repository and enter the project root.
  2. Run `env -u RUSTC_WRAPPER cargo test` to verify environment.
  3. Execute demo/live commands and inspect world evolution in viewer.

To enable LLM-driven behaviors, copy `config.example.toml` to `config.toml`, then set `AGENT_WORLD_LLM_MODEL/BASE_URL/API_KEY`.

FAQ

How is this different from typical agent demos?

It combines rule constraints, persistence, replay, and auditable evidence for long-horizon worlds.

Can I run it without LLM integration?

Yes. Rule-driven mode works without LLM, which is ideal for validating baseline mechanics first.

What is the fastest way to try it?

Run the demo to generate data, start viewer server, then open the viewer client.

Why are replay and audit emphasized?

Because reproducibility and explainability are prerequisites for serious simulation engineering.

Can I add custom modules?

Yes. The project follows a Rust + WASM extension strategy for progressive feature growth.

Where should I track project progress?

Start with README, design docs, and development logs for the latest milestones.
