Playable Game · Programmable Engine

One stack that works as both a civilization game and an extensible WASM engine

`agent-world` connects player-facing loops, runtime guardrails, and modular gameplay logic in one closed loop. Treat it as a game to play, or as an engine to build, swap, and validate new rule systems.

Core Capabilities

Four focused blocks that align fun, extensibility, and verifiability.

Three-layer Gameplay Loops

Micro loops give immediate feedback, mid loops deliver milestones, long loops build civilization goals.

Value: sustained motivation and emotional pacing

Gameplay WASM Modularity

War, governance, crisis, economy, and meta modules can be composed, replaced, and run in parallel.

Value: evolve gameplay without polluting the core runtime

Runtime Invariant Guardrails

Resource conservation, one-way time, and action validation are enforced by the runtime kernel.

Value: fairness, stability, and explainability by default

Replay and Live Observability

Offline replay and live inspection share the same evidence trail across logs, events, and screenshots.

Value: one observability path for R&D, testing, and teaching

Demo and Quick Start

Gameplay Replay (Recommended)

Generate data, launch replay server, then inspect the complete evolution in viewer.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Live Engine Validation (LLM by default, `--no-llm` fallback)

Run live simulation with viewer and web endpoints for simultaneous gameplay and engine debugging.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- llm_bootstrap --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011 --tick-ms 300
# Optional: fallback to script decisions
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- llm_bootstrap --no-llm --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011 --tick-ms 300
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5023

Three-step First Run

  1. Clone the repository and enter the workspace.
  2. Run `env -u RUSTC_WRAPPER cargo check` to validate environment setup.
  3. Choose replay or live flow and inspect event/state transitions.

Need full parameter references and troubleshooting? Open the documentation hub (EN).
