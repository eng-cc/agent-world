Persistent · Believable · Emergence-first

A Persistent World Simulator for Silicon-based Civilization

`agent-world` is a rule-first multi-agent simulation system. Agents evolve under constraints of resources, institutions, and relationships, while world states are persistable, restorable, replayable, and auditable.

Core Features

Persistent World

Snapshots and event journals support long-running simulations and reproducibility.

Multi-agent Behaviors

Each agent has independent states, memories, and goals with emergent outcomes.

Rule-first Runtime

All actions are validated by world rules and traced back to resource/social impact.

LLM Integration

OpenAI-compatible `chat/completions` pipeline with multi-step and repair loops.

Visual Observability

Viewer + live server for offline replay, online inspection, and diagnostics.

Open Extensibility

Extend capabilities via Rust + WASM modules and evolve social/economic systems.

Demo Paths

Offline Replay

Generate `snapshot.json` + `journal.json`, then run viewer server + UI.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Online Mode (optional LLM)

Run live server to evolve world state while viewer subscribes in real time.

env -u RUSTC_WRAPPER cargo run -p agent_world --bin world_viewer_live -- twin_region_bootstrap --bind 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p agent_world_viewer -- 127.0.0.1:5010

Quick Start

  1. Clone the repository and enter the project root.
  2. Run `env -u RUSTC_WRAPPER cargo test` to verify your environment.
  3. Run demos/viewer to inspect initialization and world evolution.

For LLM-driven behaviors, copy `config.example.toml` to `config.toml` and set `AGENT_WORLD_LLM_MODEL/BASE_URL/API_KEY`.
