# 启动器 Web 必填配置门控设计（2026-03-04）

- 对应需求文档: `doc/world-simulator/launcher/game-client-launcher-web-required-config-gating-2026-03-04.prd.md`
- 对应项目管理文档: `doc/world-simulator/launcher/game-client-launcher-web-required-config-gating-2026-03-04.project.md`

## 1. 设计定位
定义 web 启动器在阻断配置缺失时的门控规则、错误摘要与交互反馈，使用户在提交前就能明确知道缺失项及修复路径。

## 2. 设计结构
- 校验规则层：统一识别阻断级配置缺失与字段级问题。
- 门控反馈层：将阻断配置映射到可见错误摘要、禁用态与提示文案。
- 交互修复层：用户修改配置后即时重新评估门控状态。
- 跨端一致层：native/web 共用同一套配置问题语义，不出现分叉提示。

## 3. 关键接口 / 入口
- launcher 配置校验结果 / `ConfigIssue`
- web 启动按钮门控状态
- 配置摘要与字段级提示入口

## 4. 约束与边界
- 阻断配置必须在 UI 层可见且可诊断，不能仅依赖后端报错。
- 门控只负责可用性保护，不复制业务规则。
- native/web 必须共享配置问题语义与字段映射。
- 本阶段不扩展新的配置域，仅收敛现有阻断项反馈。

## 5. 设计演进计划
- 先固定阻断配置识别规则。
- 再补 UI 门控与错误摘要。
- 最后通过跨端回归验证按钮状态和修复路径一致。
