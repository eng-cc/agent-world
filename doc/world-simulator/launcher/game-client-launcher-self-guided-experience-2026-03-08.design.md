# 启动器自引导体验闭环设计（2026-03-08）

- 对应需求文档: `doc/world-simulator/launcher/game-client-launcher-self-guided-experience-2026-03-08.prd.md`
- 对应项目管理文档: `doc/world-simulator/launcher/game-client-launcher-self-guided-experience-2026-03-08.project.md`

## 1. 设计定位
定义启动器面向新用户的自引导闭环：通过 onboarding、任务流卡片、就地 CTA、成功配置画像、演示模式与本地引导计数，将首次使用路径从“手工摸索”收敛为可持续提示的渐进体验。

## 2. 设计结构
- 引导状态层：首次 3 步引导支持打开、跳过、完成与重置。
- 任务流层：链、游戏、页面等关键步骤形成下一步建议与 CTA。
- 轻提示层：跳过 onboarding 后继续保留轻量提示，不完全失联。
- 恢复与画像层：保存最近成功配置，支持一键恢复与演示模式。
- 洞察层：本地记录打开/跳过/完成/演示计数，形成最小产品洞察。

## 3. 关键接口 / 入口
- onboarding 向导状态
- 任务流卡片与下一步建议
- 错误卡片与一键修复 / 重试 / 修配置 CTA
- 最近成功配置画像
- 演示模式与本地引导计数

## 4. 约束与边界
- 引导必须可跳过、可重置，避免强制流程拖慢专家用户。
- CTA 需要映射到真实可执行动作，不能停留在静态说明。
- 本轮仅做本地计数，不接远端遥测。
- 自引导层不能与原有状态机分叉，需复用现有稳定动作链路。

## 5. 设计演进计划
- 先建立 onboarding 与任务流卡片骨架。
- 再补 CTA、配置画像、转账/浏览器快捷路径与术语解释。
- 最后加入演示模式和本地计数，用跨端回归验证闭环稳定性。
