# LLM 配置 TOML 风格统一（2026-03-02）设计

- 对应需求文档: `doc/world-simulator/llm/llm-config-toml-style-unification-2026-03-02.prd.md`
- 对应项目管理文档: `doc/world-simulator/llm/llm-config-toml-style-unification-2026-03-02.project.md`

## 1. 设计定位
定义 LLM 配置 TOML 风格统一设计，收敛配置命名、层级和默认值表达，降低多入口配置漂移。

## 2. 设计结构
- 配置命名层：统一字段命名、层级与注释风格。
- 默认值层：明确缺省值、覆盖顺序与环境耦合。
- 加载校验层：在读取配置时执行合法性和兼容校验。
- 文档同步层：让示例、CLI 与配置文件说明保持一致。

## 3. 关键接口 / 入口
- LLM TOML 配置字段
- 默认值与覆盖顺序
- 配置加载校验
- 配置文档示例

## 4. 约束与边界
- 统一风格不能破坏既有兼容配置。
- 配置错误必须给出明确诊断。
- 不在本专题扩展新的配置后端。

## 5. 设计演进计划
- 先梳理配置命名。
- 再统一默认值与校验。
- 最后同步示例和文档。
