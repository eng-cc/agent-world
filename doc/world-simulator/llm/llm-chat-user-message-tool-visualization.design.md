# oasis7 Simulator：LLM 用户文本输出与工具调用分区可视化（设计文档）设计

- 对应需求文档: `doc/world-simulator/llm/llm-chat-user-message-tool-visualization.prd.md`
- 对应项目管理文档: `doc/world-simulator/llm/llm-chat-user-message-tool-visualization.project.md`

## 1. 设计定位
定义 LLM 用户文本输出与工具调用分区可视化设计，让文本内容、工具调用与结果反馈在 UI 中清晰分层。

## 2. 设计结构
- 消息分区层：区分用户文本、模型文本与工具调用记录。
- 工具可视化层：展示工具参数、执行状态与结果摘要。
- 交互反馈层：让用户理解何时是对话、何时是工具执行。
- 回归验收层：验证 UI 表达与底层事件一致。

## 3. 关键接口 / 入口
- 聊天消息结构
- 工具调用展示区
- 执行状态/结果卡片
- 可视化回归用例

## 4. 约束与边界
- 文本与工具事件不得混淆展示。
- 失败工具调用需清晰暴露原因。
- 不在本专题重做整套聊天 UI 架构。

## 5. 设计演进计划
- 先分区消息类型。
- 再展示工具调用与结果。
- 最后通过回归固化交互表达。
