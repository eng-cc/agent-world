# Viewer Agent 快速定位设计

- 对应需求文档: `doc/world-simulator/viewer/viewer-agent-quick-locate.prd.md`
- 对应项目管理文档: `doc/world-simulator/viewer/viewer-agent-quick-locate.project.md`

## 1. 设计定位
定义 Viewer 右侧事件联动区的“快速定位 Agent”动作，复用现有选中与高亮链路，降低对象密集场景下手工寻找 Agent 的成本。

## 2. 设计结构
- 动作选择层：优先定位当前已选中 Agent，否则回退到字典序第一个 Agent。
- UI 入口层：在事件联动区暴露快速定位按钮，并兼容旧控件路径。
- 复用链路层：继续沿用现有选中、聚焦和高亮流程，不新增协议。
- 回归层：覆盖动作选择、按钮标签与空场景行为测试。

## 3. 关键接口 / 入口
- 右侧 `事件联动` 区按钮
- 当前选中 Agent
- Agent 列表字典序选择
- 现有选中 / 高亮 / 聚焦链路

## 4. 约束与边界
- 快速定位只触发现有选中逻辑，不创建新状态机。
- 空场景或无 Agent 时必须给出稳定可预测行为。
- 旧 UI 路径的标签与行为要保持兼容，避免回归。
- 本轮不增加新的 Viewer 协议或服务依赖。

## 5. 设计演进计划
- 先加入按钮与目标选择逻辑。
- 再复用现有聚焦/高亮链路完成定位。
- 最后补测试与标签更新，固定行为基线。
