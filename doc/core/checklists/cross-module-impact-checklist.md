# Core 跨模块变更影响检查清单（草案）

## 目的
- 为 `TASK-CORE-003` 提供标准化检查模板，覆盖设计/代码/测试/发布四类影响面。

## 适用范围
- 适用于跨两个及以上模块的需求或行为变更评审。

## 输入
- 变更对应 PRD-ID。
- 关联模块 `prd.md` / `prd.project.md`。
- 对应测试证据与发布说明。

## 检查项
- 设计影响：模块边界、术语、接口是否一致。
- 代码影响：受影响 crate/脚本/配置是否识别完整。
- 测试影响：required 门禁与专项回归是否覆盖。
- 发布影响：风险、回滚条件、证据包是否齐全。

## 阻断条件
- 任一关键模块无 PRD-ID 追溯映射。
- 任一高风险改动无最小 required 证据。

## 回滚策略
- 出现阻断条件时暂停发布结论，回退到“补文档/补测试”状态。
