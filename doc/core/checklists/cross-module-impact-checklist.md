# Core 跨模块变更影响检查清单

审计轮次: 3

## 目的
- 为 `TASK-CORE-003` 提供可执行检查模板，覆盖设计/代码/测试/发布四类影响面。

## 适用范围
- 适用于跨两个及以上模块的需求、行为或接口变更评审。

## 输入
- 变更对应 PRD-ID 列表。
- 关联模块 `prd.md` / `prd.project.md` 与任务状态。
- 对应测试证据与发布说明（命令、日志、产物路径）。

## 使用规则
- 检查结果仅允许使用 `✔ / ❌ / ⚠ / N/A`。
- 标记 `N/A` 必须写明“不适用原因 + 替代验证方式 + 评审人”。
- 出现 `❌` 或未关闭的 `⚠`，必须在“整改项清单”登记“负责人 + 截止时间”。
- 若确认无需整改，必须在“特殊情况备注”登记“原因 + 风险 + 临时缓解 + 复审日期”。

- 任一关键模块缺失 PRD-ID 追溯映射。

- 存在 `❌` 且未登记可执行整改项。
