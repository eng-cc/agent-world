# Capture Viewer Frame（Agent UI截图闭环调试脚本）设计

- 对应需求文档: `doc/scripts/viewer-tools/capture-viewer-frame.prd.md`
- 对应项目管理文档: `doc/scripts/viewer-tools/capture-viewer-frame.project.md`

## 1. 设计定位
定义 viewer 工具脚本专题设计，统一抓帧、材质检查、细节优化与纹理质检工具链。

## 2. 设计结构
- 工具入口层：明确抓帧、纹理检查与艺术校准脚本入口。
- 分析校验层：围绕材质、框架、视觉细节和可辨识性建立检查点。
- 产物输出层：定义截图、标注、分析结果与归档方式。
- 演进维护层：沉淀工具脚本合理化与后续优化路径。

## 3. 关键接口 / 入口
- viewer 工具脚本入口
- 截图/分析输出产物
- 材质与视觉校验点
- 工具维护与回归说明

## 4. 约束与边界
- 工具脚本应优先服务 viewer 闭环调试。
- 输出产物需可复核、可归档。
- 不在本专题扩展新的渲染框架。

## 5. 设计演进计划
- 先固定工具入口与产物格式。
- 再补分析校验规则。
- 最后固化合理化与维护路径。
