# GitHub Pages 对外展示站点设计

## 目标
- 在仓库内新增一个可公开访问的 GitHub Pages 站点，用于对外介绍 `agent-world`。
- 让外部访客在 1 分钟内理解项目定位、核心能力、演示入口与参与方式。
- 采用低维护成本方案，确保后续内容更新可由仓库协作流程持续驱动。
- 提供中文与英文双版本入口，满足对外展示的国际化基础需求。

## 范围
- **范围内**：
  - 提供首版对外展示站点（优先单页 Landing Page，可扩展多页）。
  - 提供中英文双版页面（中文默认页 + 英文入口页）。
  - 展示内容包含：项目愿景、核心特性、运行演示、快速开始、文档入口、路线图摘要。
  - 支持移动端与桌面端的基础自适应。
  - 接入 GitHub Actions 自动部署到 GitHub Pages。
- **范围外**：
  - 在线运行仿真/Viewer（仅展示截图、短 GIF 或录屏片段）。
  - 引入后端服务、账号系统、评论系统。
  - 将现有全部技术文档迁移为完整文档站。

## 方案对比
### 方案 A：纯静态页面（HTML/CSS/少量 JS）
- 优点：零额外构建依赖、部署链路最短、与当前 Rust 工程解耦。
- 缺点：后续页面规模变大时，模板复用能力较弱。

### 方案 B：静态站点生成器（如 VitePress/MkDocs）
- 优点：主题与导航体系成熟，适合后续扩展文档门户。
- 缺点：增加 Node/Python 工具链，CI 与维护复杂度提升。

### 结论
- 首版采用**方案 A（纯静态）**。
- 当页面数量与协作人数明显增长后，再评估迁移到静态站点生成器。

## 接口 / 数据
- 页面目录（建议）：
  - `site/index.html`（中文页）
  - `site/en/index.html`（英文页）
  - `site/assets/styles.css`
  - `site/assets/app.js`
  - `site/assets/images/*`
- 语言策略（首版）：
  - 默认入口为中文页 `index.html`。
  - 首次访问时，若浏览器首选语言为英文（`en*`），自动跳转到 `en/`。
  - 用户点击语言切换后，记录手动选择并停止自动重定向（尊重用户选择）。
  - 页面顶部语言入口使用“最右侧地球图标按钮”，点击后展示语言下拉菜单。
- 前端本地存储键：
  - `agent_world_pages_lang_redirect_done_v1`
  - `agent_world_pages_lang_manual_choice_v1`
- 部署工作流（建议新增）：
  - `.github/workflows/pages.yml`
  - 触发：`push` 到 `main`（限制 `site/**` 与 workflow 相关路径）。
  - 产物：上传 `site/` 目录并部署到 GitHub Pages。
- 内容来源：
  - 主来源：`README.md` 与 `doc/world-simulator.md` 的公开信息。
  - 展示素材：通过现有 viewer 截图流程生成，再挑选到 `site/assets/images/`。
- 链接策略：
  - 使用相对路径资源，兼容 `https://<org>.github.io/<repo>/` 项目页路径前缀。

## 里程碑
- **M1（方案与文案）**：确定信息架构、首页模块与首版文案。
- **M2（页面实现）**：完成静态页面结构、样式与响应式布局。
- **M2.1（双语能力）**：补齐英文页面与中英文切换入口。
- **M3（自动部署）**：接入 `pages.yml`，实现 push 自动发布。
- **M4（素材增强，可选）**：补充演示 GIF、关键指标卡片与 FAQ。

## 风险
- **内容过时**：主项目演进快，展示站文案可能滞后。
  - 缓解：在项目任务中增加“发布前文案核对”检查点。
- **素材体积过大**：截图/GIF 过大影响首屏加载。
  - 缓解：统一压缩策略并控制单素材体积。
- **路径与部署细节易错**：项目页与组织页路径差异可能导致资源 404。
  - 缓解：本地按子路径预览 + CI 后快速 smoke check。
- **双语内容漂移**：中英文页面长期演进可能出现信息不一致。
  - 缓解：将双语更新加入同一任务清单与发布前检查项。
- **自动跳转体验偏差**：部分用户不希望按浏览器语言自动跳转。
  - 缓解：仅首访触发且支持手动切换后记忆，不再强制跳转。

## 验收标准（首版）
- 外部访问地址可打开，首页内容完整且无明显样式错乱。
- 首页包含“项目定位、核心能力、演示截图、快速开始、文档入口”五类信息。
- 中文页与英文页均可访问，且页面提供相互切换入口。
- 英文浏览器首访根路径可自动进入英文页，手动切换语言后保持用户选择。
- 任意改动 `site/**` 后可自动触发部署。
- README 增加对外展示站入口链接。
