# GitHub Pages 首页“游戏优先”分层重构（2026-02-25）设计文档

- 对应需求文档: `doc/site/github-pages/github-pages-game-first-home-2026-02-25.prd.md`
- 对应项目管理文档: `doc/site/github-pages/github-pages-game-first-home-2026-02-25.project.md`

## 1. 设计定位
定义 GitHub Pages 站点专题设计，统一首页信息架构、视觉层、内容同步、转化入口与发布流水线。

## 2. 设计结构
- 信息架构层：明确首页、展示页、下载入口与文档镜像的组织结构。
- 视觉交互层：收敛 hero、CTA、动效、截图与视觉精修策略。
- 内容同步层：对齐中英内容、SEO 元信息、文档镜像与发布状态。
- 发布运营层：把 release 下载、质量门禁和用户转化路径串成闭环。

## 3. 关键接口 / 入口
- GitHub Pages 页面入口
- hero/CTA/视觉组件
- 内容同步与 SEO 配置
- release/download 入口与质量门禁

## 4. 约束与边界
- 站点改造需保持对外口径一致。
- 内容同步不得破坏多语言和镜像可达性。
- 不在本专题引入新的站点框架栈。

## 5. 设计演进计划
- 先冻结信息架构与核心视觉层。
- 再补内容同步、SEO 和下载流水线。
- 最后固化质量门禁与转化回归。
