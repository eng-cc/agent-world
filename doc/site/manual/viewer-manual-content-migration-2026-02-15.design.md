# Viewer 使用手册内容搬迁（2026-02-15）设计文档

- 对应需求文档: `doc/site/manual/viewer-manual-content-migration-2026-02-15.prd.md`
- 对应项目管理文档: `doc/site/manual/viewer-manual-content-migration-2026-02-15.project.md`

## 1. 设计定位
定义站点手册专题设计，统一静态文档站、CN/EN 手册迁移与长期维护约定。

## 2. 设计结构
- 手册承载层：明确静态站对手册内容、语言版本与入口的组织方式。
- 迁移编排层：把原有 viewer/manual 内容迁移到静态站可维护结构。
- 发布同步层：确保手册更新与站点发布、导航和互链同步。
- 维护治理层：沉淀手册更新责任、版本约定与质量检查。

## 3. 关键接口 / 入口
- 静态文档站入口
- CN/EN 手册内容目录
- 手册迁移与导航映射
- 手册发布与维护约定

## 4. 约束与边界
- 手册迁移不得破坏现有可访问路径。
- 语言版本需保持结构和口径一致。
- 不在本专题扩展新的内容管理系统。

## 5. 设计演进计划
- 先固定静态手册结构。
- 再补内容迁移与导航映射。
- 最后固化发布同步与维护治理。
