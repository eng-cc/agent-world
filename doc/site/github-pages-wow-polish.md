# GitHub Pages 惊艳化精修（二期）设计

## 目标
- 将当前展示页从“信息可读”升级为“科技感强、叙事完整、可被记住”的项目门面。
- 让首次访客在 30 秒内被视觉与价值主张吸引，在 3 分钟内完成“理解能力边界 + 产生信任 + 愿意尝试”。
- 保持纯静态部署（GitHub Pages），在不引入复杂构建链路前提下实现高质量视觉与交互。
- 中英文双语内容保持一致演进，避免“中文丰富、英文简略”的体验落差。

## 范围
- **范围内**
  - 视觉系统升级：统一设计 Token、科技感背景、分层光效、卡片与按钮状态体系。
  - 内容深度升级：首页模块从“简介型”扩展到“证据型”（架构、能力矩阵、场景、路线、FAQ、贡献路径）。
  - 交互升级：首屏微动效、滚动显隐、数字计数、时间线、对比切换等轻量交互。
  - 信任建设：补充可验证证据（命令示例、截图、测试状态、版本节点）。
  - SEO 与传播：完善 meta、OpenGraph、结构化文案与分享图。
  - 质量保障：性能、可访问性、移动端可用性、Playwright 截图回归。
- **范围外**
  - 接入后端接口或在线实时运行环境。
  - 引入复杂前端框架与构建系统（保持 HTML/CSS/JS 静态方案）。
  - 完整文档站替代（本次仍聚焦 Showcase）。

## 设计原则
- **第一屏先打动**：先感性冲击，再理性证明。
- **每屏一个核心问题**：减少冗余句子，提升信息密度。
- **视觉为叙事服务**：动效不喧宾夺主，优先可读性和速度。
- **证据优先于口号**：每个“能力点”至少配一个可核验事实。
- **低维护可持续**：内容模块化、数据化，后续更新成本可控。

## 体验蓝图（信息架构）
> 在当前中英文双页基础上扩展，保持目录与锚点一致。

1. `Hero`（价值主张 + 背景动效 + 主 CTA）
2. `Why Agent-World`（三大差异化能力）
3. `核心能力矩阵`（能力 × 场景 × 价值）
4. `架构剖面`（Runtime / Simulator / Viewer / LLM 接入）
5. `实战场景`（2~3 个高价值应用故事）
6. `演示与命令`（离线/在线双路径，快速复制）
7. `里程碑时间线`（已完成 + 近期计划）
8. `质量与可信度`（测试、可观测、审计能力）
9. `FAQ`（上手成本、资源要求、LLM 配置）
10. `参与贡献`（Issue、PR、讨论入口）
11. `页脚导航`（文档、仓库、许可证、语言切换）

## 接口 / 数据

### 目录与文件组织（建议）
- 页面：
  - `site/index.html`
  - `site/en/index.html`
- 样式：
  - `site/assets/styles.css`（新增 Token 分层）
  - `site/assets/styles.motion.css`（可选，动效分离）
- 脚本：
  - `site/assets/app.js`（交互入口）
  - `site/assets/modules/scroll-reveal.js`
  - `site/assets/modules/counter.js`
  - `site/assets/modules/theme-surface.js`
- 内容数据（降低维护成本）：
  - `site/data/metrics.json`
  - `site/data/timeline.json`
  - `site/data/faq.zh.json`
  - `site/data/faq.en.json`
- 素材：
  - `site/assets/images/screenshots/*`
  - `site/assets/images/og-cover.png`

### 视觉 Token（CSS Variables）
- 颜色层级：`--bg-0 ~ --bg-4`、`--brand-primary`、`--brand-accent`。
- 光效层级：`--glow-soft`、`--glow-strong`。
- 字体层级：`--font-display`、`--font-body`、`--font-mono`。
- 间距/圆角/阴影统一 Token，减少局部硬编码。

### 动效与可访问性策略
- 所有动效支持 `prefers-reduced-motion` 降级。
- 所有交互控件必须支持键盘访问与可见焦点。
- 首屏背景动效限制在 CSS/Canvas 轻量实现，禁止大体积视频作为首屏依赖。

### 文案数据策略
- 核心结构中英文对齐（同锚点、同模块顺序）。
- 可变内容（时间线、FAQ、指标）尽量外置 JSON，减少多页手工同步成本。

## 里程碑

### M1：内容与视觉基线（1~2 天）
- 完成信息架构重排与双语文案骨架。
- 交付新版视觉 Token 与组件样式基线（卡片/按钮/section）。

### M2：首屏与叙事升级（1~2 天）
- 完成 Hero 科技感改造（背景层、核心主张、CTA 优化）。
- 新增能力矩阵 + 架构剖面模块。

### M3：深度内容补齐（1~2 天）
- 新增实战场景、里程碑时间线、FAQ、质量可信度模块。
- 引入真实截图/GIF（压缩后）并完成中英文一致更新。

### M4：交互与体验打磨（1 天）
- 加入滚动显隐、计数动画、导航激活态、锚点定位优化。
- 完成移动端适配细节（折叠菜单、可点击区域、长文断行）。

### M5：发布质量门禁（0.5~1 天）
- Lighthouse（Mobile）Performance/Accessibility/Best Practices/SEO 目标 >= 90。
- Playwright 多视口截图回归 + 链接检查 + 资源体积检查。

## 风险
- **视觉复杂度过高导致性能下降**
  - 缓解：动效默认轻量，优先 CSS；关键资源懒加载与压缩。
- **双语内容维护成本上升**
  - 缓解：结构数据化、同任务双语同步、发布前双语 checklist。
- **素材不足导致“科技感空心化”**
  - 缓解：提前规划截图脚本与产出清单，先上静态图再迭代 GIF。
- **信息过多导致阅读负担**
  - 缓解：模块分层（总览→细节→跳转），每屏只保留一个主结论。

## 验收标准
- 首屏在 5 秒内传达：是什么、为什么重要、如何马上体验。
- 页面完整覆盖“定位、能力、架构、场景、证据、参与方式”。
- 中英文页面结构一致，且语言切换路径无断链。
- 页面视觉显著提升，但交互不卡顿、无明显跳动错位。
- 部署后可在 GitHub Pages 项目路径正常访问，关键资源无 404。

## 实施建议（优先级）
- **P0（必须）**：Hero 升级、能力矩阵、架构剖面、实战场景、FAQ、移动端优化。
- **P1（应做）**：时间线、计数动画、分享图、SEO 元信息、截图回归。
- **P2（可选）**：更高级视觉粒子效果、主题切换、微交互增强。
