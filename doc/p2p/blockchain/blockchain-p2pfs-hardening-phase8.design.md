# 区块链 + P2PFS 硬改造 Phase 8 设计

- 对应需求文档: `doc/p2p/blockchain/blockchain-p2pfs-hardening-phase8.prd.md`
- 对应项目管理文档: `doc/p2p/blockchain/blockchain-p2pfs-hardening-phase8.project.md`

## 1. 设计定位
定义链路收紧后的配置、校验与恢复收口方案：把前几阶段的 signer/白名单/策略治理进一步固化到生产可复用入口中。

## 2. 设计结构
- 配置收口层：把前置阶段新增的签名和白名单策略统一纳入活跃配置入口。
- 恢复语义层：确保恢复、同步与重启路径都共享同一安全约束。
- 运维可诊断层：对拒绝原因、配置缺失和策略冲突给出稳定口径。
- 阶段验收层：通过回归测试确认多阶段安全治理没有回退。

## 3. 关键接口 / 入口
- 活跃配置入口
- 恢复/同步/重启共享策略
- 稳定错误口径
- 阶段验收测试

## 4. 约束与边界
- 不额外扩展新的密码学算法或底层网络协议。
- 重点是把前序阶段成果收口，而非新增大规模功能。
- 恢复路径必须与实时路径保持同一安全口径。
- 诊断口径要利于运维排障，而不是只输出内部断言。

## 5. 设计演进计划
- 先盘点前序安全策略入口。
- 再统一恢复/同步/实时路径语义。
- 最后用回归验证 phase8 收口。
