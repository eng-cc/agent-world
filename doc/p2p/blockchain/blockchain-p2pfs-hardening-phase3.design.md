# 区块链 + P2PFS 硬改造 Phase 3 设计

- 对应需求文档: `doc/p2p/blockchain/blockchain-p2pfs-hardening-phase3.prd.md`
- 对应项目管理文档: `doc/p2p/blockchain/blockchain-p2pfs-hardening-phase3.project.md`

## 1. 设计定位
定义区块链 + P2PFS 硬改造第三阶段方案：继续扩展跨 crate 的签名治理、同步路径和关键安全边界，为后续 membership 和信任根治理铺路。

## 2. 设计结构
- 签名治理层：扩展跨链路消息的签名验证与兼容策略。
- 同步接线层：把新增安全语义接入发布、同步与恢复路径。
- 测试兜底层：针对新签名治理和同步路径补齐 targeted 回归。
- 阶段收口层：保持 strict PRD 重排后的约束和原文技术边界一致。

## 3. 关键接口 / 入口
- 跨 crate 签名治理入口
- 发布/同步路径
- strict PRD 约束点映射
- 回归测试矩阵

## 4. 约束与边界
- 保留原文技术边界，不在 Design 中额外扩 scope。
- 兼容阶段仍需支持历史链路，不能一次切断旧路径。
- 安全增强优先走渐进开关，不做无保护硬切换。
- 文档治理目标是把设计权威源独立出来，而非重写业务目标。

## 5. 设计演进计划
- 先冻结原文约束与阶段目标。
- 再补签名治理与同步接线的结构视图。
- 最后通过文档互链和回归口径收口 phase3。
