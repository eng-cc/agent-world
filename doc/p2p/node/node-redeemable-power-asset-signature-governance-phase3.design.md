# oasis7 Runtime：可兑现节点资产与电力兑换闭环（三期真实签名与治理闭环）设计

- 对应需求文档: `doc/p2p/node/node-redeemable-power-asset-signature-governance-phase3.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-redeemable-power-asset-signature-governance-phase3.project.md`

## 1. 设计定位
定义可兑现节点资产与电力兑换闭环三期真实签名与治理闭环方案，把签名凭证真正纳入治理决策与执行门禁。

## 2. 设计结构
- 真实签名层：接入生产可用的真实签名凭证而非占位路径。
- 治理决策层：让兑换/资产操作受治理审批与阈值约束。
- 执行门禁层：无有效签名或治理决议时禁止推进关键状态。
- 运维复核层：输出签名与治理决策的审计证据。

## 3. 关键接口 / 入口
- 真实签名凭证入口
- 治理审批状态
- 执行门禁检查
- 签名/治理审计视图

## 4. 约束与边界
- 治理结果必须可追溯到签名主体与决议记录。
- 门禁优先保证安全，不做静默放行。
- 不在本专题扩展更大范围的治理经济机制。

## 5. 设计演进计划
- 先替换为真实签名路径。
- 再接治理审批与执行门禁。
- 最后固化审计与回归。
