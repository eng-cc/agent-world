# Agent World Runtime：节点基础在线时长奖励设计

- 对应需求文档: `doc/p2p/node/node-uptime-base-reward.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-uptime-base-reward.project.md`

## 1. 设计定位
定义节点基础在线时长奖励方案，用稳定在线时长作为基础奖励输入并与更高阶贡献奖励分层。

## 2. 设计结构
- 在线采集层：记录节点在线时长与可用性窗口。
- 基础奖励计算层：按在线时长累计基础奖励。
- 分层叠加层：与复制、执行等高阶奖励口径协同。
- 观测审计层：输出在线奖励明细与异常扣减原因。

## 3. 关键接口 / 入口
- 在线时长指标
- 基础奖励计算入口
- 奖励分层规则
- 在线奖励审计视图

## 4. 约束与边界
- 基础奖励计算需抗短时抖动。
- 在线奖励不得与其他奖励重复计量。
- 不在本专题实现复杂信誉评分。

## 5. 设计演进计划
- 先固定在线时长统计口径。
- 再接基础奖励计算。
- 最后联动总奖励体系回归。
