# oasis7 Runtime：节点贡献积分激励设计

- 对应需求文档: `doc/p2p/node/node-contribution-points.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-contribution-points.project.md`

## 1. 设计定位
定义节点贡献积分激励主模型，统一贡献事件、积分累计、结算入口与奖励约束。

## 2. 设计结构
- 贡献采集层：采集在线、复制、执行与协作等贡献事件。
- 积分计算层：按规则累计积分并处理去重、衰减与封顶。
- 奖励映射层：把积分映射到奖励结算与治理口径。
- 观测审计层：输出节点贡献明细、汇总结果与回归证据。

## 3. 关键接口 / 入口
- 贡献事件模型
- 积分累计入口
- 奖励映射规则
- 贡献审计视图

## 4. 约束与边界
- 积分口径需与奖励结算口径保持一致。
- 重复贡献不得被重复计分。
- 不在本专题直接定义跨链资产发行细节。

## 5. 设计演进计划
- 先冻结贡献事件与积分规则。
- 再接运行时闭环和多节点回归。
- 最后联动奖励与治理专题收口。
