# oasis7 Runtime：节点贡献积分运行时闭环设计

- 对应需求文档: `doc/p2p/node/node-contribution-points-runtime-closure.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-contribution-points-runtime-closure.project.md`

## 1. 设计定位
定义节点贡献积分在运行时中的落地闭环，确保贡献事件从世界运行时进入积分和奖励主路径。

## 2. 设计结构
- 运行时接线层：把贡献事件接入 runtime 主循环。
- 状态更新层：在节点状态中维护积分累计与派生指标。
- 奖励桥接层：把积分结果桥接到后续奖励结算入口。
- 回归层：围绕单节点运行时路径执行定向验证。

## 3. 关键接口 / 入口
- runtime 贡献事件入口
- 节点积分状态
- 奖励桥接接口
- 运行时回归用例

## 4. 约束与边界
- 运行时接线不得破坏既有节点主循环稳定性。
- 积分状态更新需具备幂等与可重放性。
- 本专题聚焦单节点运行时闭环，不覆盖多节点协作验证。

## 5. 设计演进计划
- 先完成 runtime 接线。
- 再固化状态更新与奖励桥接。
- 最后执行定向闭环回归。
