# Node 共识签名身份绑定与复制摄取硬化设计

- 对应需求文档: `doc/p2p/node/node-consensus-signer-binding-replication-hardening.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-consensus-signer-binding-replication-hardening.project.md`

## 1. 设计定位
定义 Node 共识签名身份与复制摄取链路的联合硬化方案，确保消息签名身份、复制来源与摄取约束保持一致。

## 2. 设计结构
- 签名绑定层：共识消息与节点身份、公钥配置建立稳定绑定。
- 复制摄取层：复制网络只接受符合身份与来源约束的数据。
- 校验拒收层：异常签名、错配身份和非法来源在摄取前阻断。
- 审计回写层：把关键签名与摄取决策暴露给回归与运维核查。

## 3. 关键接口 / 入口
- 节点签名身份配置
- 共识消息验签入口
- 复制摄取校验入口
- 审计日志/拒收原因

## 4. 约束与边界
- 身份绑定策略必须兼容现有节点引导方式。
- 复制摄取异常优先拒收，不做静默降级。
- 不在本专题引入新的治理角色模型。

## 5. 设计演进计划
- 先收敛 signer 与节点身份映射。
- 再加固 replication ingest 校验。
- 最后补回归与文档台账。
