# oasis7 Runtime：PoS 时间锚定控制面参数与可观测口径对齐设计

- 对应需求文档: `doc/p2p/node/node-pos-time-anchor-control-plane-alignment-2026-03-07.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-pos-time-anchor-control-plane-alignment-2026-03-07.project.md`

## 1. 设计定位
定义 PoS 时间锚定控制面参数与可观测口径对齐方案，确保配置、运行状态与运维读数使用同一时间语义。

## 2. 设计结构
- 控制面参数层：统一时间锚定相关配置字段与默认值。
- 运行状态层：把实际生效的时间锚点、偏差和窗口暴露出来。
- 可观测口径层：让 metrics/logs/dashboard 使用统一命名与解释。
- 治理回写层：把偏差、调整和发布口径沉淀到文档。

## 3. 关键接口 / 入口
- 时间锚定配置参数
- 运行时生效状态
- metrics/logs 仪表项
- 控制面校验入口

## 4. 约束与边界
- 配置口径与运行口径不得出现双重命名。
- 可观测字段变更需保持向后可审计。
- 不在本专题新增复杂的自动校时治理。

## 5. 设计演进计划
- 先统一控制面参数。
- 再对齐运行状态与观测项。
- 最后回写运维与治理文档。
