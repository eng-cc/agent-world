# Agent World Runtime：节点贡献积分多节点闭环测试设计

- 对应需求文档: `doc/p2p/node/node-contribution-points-multi-node-closure-test.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-contribution-points-multi-node-closure-test.project.md`

## 1. 设计定位
定义节点贡献积分在多节点环境下的闭环测试设计，验证多节点贡献采集、汇总与奖励映射的一致性。

## 2. 设计结构
- 多节点场景层：构造 leader/follower/replica 等典型节点协作场景。
- 贡献汇总层：验证多来源贡献事件在集群中被正确统计。
- 一致性校验层：比对节点本地视图与全局汇总结果。
- 回归归档层：沉淀多节点闭环测试矩阵与失败签名。

## 3. 关键接口 / 入口
- 多节点测试拓扑
- 贡献汇总断言
- 全局一致性校验
- 失败签名归档

## 4. 约束与边界
- 测试以验证贡献积分闭环为主，不扩展性能压测目标。
- 多节点重复事件必须有幂等守卫。
- 失败签名需要可复现、可追踪。

## 5. 设计演进计划
- 先搭建多节点贡献场景。
- 再补充汇总与一致性断言。
- 最后固定为回归矩阵。
