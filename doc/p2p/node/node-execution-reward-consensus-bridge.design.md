# oasis7 Runtime：节点执行桥接与奖励共识触发闭环设计

- 对应需求文档: `doc/p2p/node/node-execution-reward-consensus-bridge.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-execution-reward-consensus-bridge.project.md`

## 1. 设计定位
定义节点执行结果、奖励结算与共识触发之间的桥接方案，让执行贡献可以进入奖励主路径并受共识保护。

## 2. 设计结构
- 执行采集层：收集可计奖的执行结果与验证结论。
- 桥接转换层：把执行结果转换为奖励计算与共识输入。
- 共识触发层：在合适高度/批次触发奖励共识与状态推进。
- 回写观测层：沉淀执行到奖励的映射证据与审计记录。

## 3. 关键接口 / 入口
- 执行结果入口
- 奖励桥接模型
- 共识触发点
- 执行-奖励审计记录

## 4. 约束与边界
- 奖励触发必须以共识确认的执行结果为准。
- 桥接层不得引入双重计奖。
- 不在本专题重写完整奖励经济模型。

## 5. 设计演进计划
- 先收敛执行结果与奖励输入模型。
- 再接共识触发点。
- 最后补回归与运维口径。
