# oasis7 Runtime：节点奖励运行时生产化加固（Phase 1）设计文档

- 对应需求文档: `doc/p2p/node/node-reward-runtime-production-hardening-phase1.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-reward-runtime-production-hardening-phase1.project.md`

## 1. 设计定位
定义节点奖励运行时生产化加固第一阶段方案，先把奖励运行时的关键状态、结算入口和失败守卫补齐。

## 2. 设计结构
- 奖励状态层：收敛奖励计算、待结算与已结算状态。
- 运行时守卫层：为奖励触发、重复执行和异常恢复增加保护。
- 结算接线层：把奖励结果接入运行时主路径。
- 生产回归层：围绕关键奖励路径执行 required-tier 回归。

## 3. 关键接口 / 入口
- 奖励运行时状态
- 奖励触发入口
- 结算接线点
- 奖励回归矩阵

## 4. 约束与边界
- 优先保证奖励正确性与幂等。
- 异常恢复不得造成重复发奖。
- 本阶段不追求完整奖励体系重构。

## 5. 设计演进计划
- 先加固状态与守卫。
- 再补结算接线。
- 最后执行生产化回归。
