# oasis7 Runtime：节点存储系统奖励池设计

- 对应需求文档: `doc/p2p/node/node-storage-system-reward-pool.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-storage-system-reward-pool.project.md`

## 1. 设计定位
定义节点存储系统奖励池方案，把存储贡献、奖励池余额与分配规则统一到可审计的系统闭环。

## 2. 设计结构
- 奖励池建模层：定义奖励池余额、来源与消耗规则。
- 存储贡献映射层：把存储容量、可用性和复制贡献映射到奖励池分配。
- 结算分发层：按周期或事件触发奖励池分发。
- 审计治理层：保留池子变化、分配记录与治理限制。

## 3. 关键接口 / 入口
- 奖励池台账
- 存储贡献指标
- 奖励池分发入口
- 池子审计记录

## 4. 约束与边界
- 奖励池余额变化必须可追溯。
- 分配规则需与节点贡献口径一致。
- 不在本专题定义完整代币经济。

## 5. 设计演进计划
- 先固定奖励池模型。
- 再接存储贡献映射。
- 最后补结算与审计。
