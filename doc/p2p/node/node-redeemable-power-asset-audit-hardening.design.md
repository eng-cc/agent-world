# Agent World Runtime：可兑现节点资产与电力兑换闭环（二期审计与签名加固）设计

- 对应需求文档: `doc/p2p/node/node-redeemable-power-asset-audit-hardening.prd.md`
- 对应项目管理文档: `doc/p2p/node/node-redeemable-power-asset-audit-hardening.project.md`

## 1. 设计定位
定义可兑现节点资产与电力兑换闭环二期审计与签名加固方案，补强资产变更、兑换操作与审核链路的可信度。

## 2. 设计结构
- 审计留痕层：记录资产与兑换操作的关键审计字段。
- 签名加固层：为关键申请、审批与结算动作增加签名约束。
- 异常阻断层：对缺失审计、签名错配和违规流转直接阻断。
- 回归复核层：沉淀审计与签名场景的回归矩阵。

## 3. 关键接口 / 入口
- 资产审计日志
- 兑换签名校验入口
- 异常阻断规则
- 审计回归用例

## 4. 约束与边界
- 审计字段需覆盖关键状态变化。
- 签名加固不能破坏已有兑现主路径可用性。
- 不在本专题引入完整链上治理框架。

## 5. 设计演进计划
- 先补审计字段。
- 再加固签名校验。
- 最后执行定向复核回归。
