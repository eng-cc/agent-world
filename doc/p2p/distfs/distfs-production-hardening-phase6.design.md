# DistFS 生产化增强 Phase 6 设计

- 对应需求文档: `doc/p2p/distfs/distfs-production-hardening-phase6.prd.md`
- 对应项目管理文档: `doc/p2p/distfs/distfs-production-hardening-phase6.project.md`

## 1. 设计定位
定义 DistFS challenge 有状态调度的自适应失败退避方案：在持续失败时降低无效探测和 I/O 抖动，同时保持状态文件兼容。

## 2. 设计结构
- 失败退避层：根据失败情况引入 backoff，减少持续失败时的探测频率。
- 预算上限层：限制单轮挑战预算，避免高密度 blob 场景放大负载。
- 状态兼容层：新旧 probe 状态文件演进保持线上可升级。
- 观测验证层：通过回归确认退避不会破坏正常路径。

## 3. 关键接口 / 入口
- adaptive backoff 策略
- 单轮挑战预算上限
- 状态文件兼容演进
- probe 调度回归测试

## 4. 约束与边界
- 退避优化只针对稳定性，不改变 challenge 基本语义。
- 状态文件演进必须向后兼容，避免线上升级阻断。
- 预算上限是控制负载，不是削弱覆盖目标的永久替代。
- 退避策略要避免把临时抖动放大为长期沉默。

## 5. 设计演进计划
- 先定义失败退避和预算上限。
- 再接状态兼容加载。
- 最后通过探测回归验证稳定性收益。
