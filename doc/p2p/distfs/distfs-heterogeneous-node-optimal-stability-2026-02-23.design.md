# DistFS 异构节点最优稳定性设计

- 对应需求文档: `doc/p2p/distfs/distfs-heterogeneous-node-optimal-stability-2026-02-23.prd.md`
- 对应项目管理文档: `doc/p2p/distfs/distfs-heterogeneous-node-optimal-stability-2026-02-23.project.md`

## 1. 设计定位
定义 DistFS 在异构节点条件下的稳定性优化方案：根据节点能力差异收敛复制、拉取和恢复策略，降低单一最强节点假设。

## 2. 设计结构
- 节点分层层：根据能力和角色区分不同节点承担的复制/服务职责。
- 稳定性策略层：调整复制、拉取和回放路径以适配异构节点性能差异。
- 失败恢复层：针对慢节点、弱节点和部分失联节点建立更稳的降级路径。
- 验证收口层：通过异构场景回归确认系统在多类节点组合下仍可用。

## 3. 关键接口 / 入口
- 异构节点能力分层
- 复制/拉取策略
- 恢复与降级路径
- 异构场景回归

## 4. 约束与边界
- 不引入新的底层编码协议或负载均衡控制面。
- 重点是稳定性和策略，不是追求理论最优调度。
- 不能再隐含单一最强节点提供完整覆盖。
- 设计需与 no-single-full-node 假设保持一致。

## 5. 设计演进计划
- 先冻结异构节点分层假设。
- 再补复制/恢复策略。
- 最后通过场景回归收口稳定性目标。
