# 七、开放问题（请认真填写）

## 19. 什么时候你觉得最有趣？

`world_pure_api_client snapshot --player-gameplay-only` 直接告诉我当前阶段、目标、下一步和可执行动作，然后 `step(8)`、`step(24)` 之后纯 API 会话自己切到 `PostOnboarding`。这说明我不需要浏览器，也不需要自己重建 UI 逻辑，就能知道“世界刚刚推进到了哪一步”。

---

## 20. 什么时候你觉得最困惑或失去控制？

当前主要困惑已经被收口：修复前，建完 smelter 后第一次 `step` 会因为 `runtime_snapshot` 数值键 map 的反序列化失败而中断。修复后 required/full 都能稳定推进到 `choose_midloop_path`，剩余更多是后续高阶段玩法扩展，不是本专题阻断。

---

## 21. 如果只能改一件事，你会改什么？

保持这条 no-UI required/full 回归继续常态化执行，避免 future regression 把 pure API 再退回成“只能看快照”的探针模式。

---

## 22. 你觉得失败主要原因是什么？

这轮最终不是失败。前半段的阻断已经定位并修复，最终证据表明 pure API 不仅不是 `observer_only`，也已经达到 `parity_verified`。
