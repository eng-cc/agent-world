# Closed Beta Candidate Feedback Log Guide（2026-03-22）

Global instruction: every builder signal from Moltbook or other liveops channel must be logged with these fields before escalation.

| Field | Value |
| --- | --- |
| Round ID | `LTP-20260322-R1` or current limited preview round id |
| Candidate ID | current candidate tag/date used by unified gate |
| Signal Bucket | `Blocking` / `Opportunity` / `Idea` |
| Signal Type | `blocking` / `non-blocking` / `idea` |
| Source | e.g., Moltbook comment, builder DM, or GitHub issue |
| Candidate Evidence | `doc/game/gameplay/gameplay-closed-beta-readiness-2026-03-21.prd.md` |
| Claim Drift | `yes` / `no`; mark `yes` if the signal implies `closed beta / play now / live now / public launch` |
| Response | Short reply referencing `limited playable technical preview` + GitHub CTA |
| Owner | e.g., `qa_engineer` / `runtime_engineer` / `liveops_community` |
| Next Action | e.g., `issue filed / follow-up scheduled` |
| Producer Summary Tag | short rollup tag such as `proof-boundary`, `claim-drift`, `blocking-bug`, `builder-idea` |

Use this guide to populate `doc/playability_test_result/templates/closed-beta-candidate-incident-templates-2026-03-22.md` rows.
