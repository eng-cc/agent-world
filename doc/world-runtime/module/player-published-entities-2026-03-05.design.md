# oasis7 Runtime：玩家发布制成品的 WASM 模块与 Profile 治理闭环（2026-03-05）设计

- 对应需求文档: `doc/world-runtime/module/player-published-entities-2026-03-05.prd.md`
- 对应项目管理文档: `doc/world-runtime/module/player-published-entities-2026-03-05.project.md`

## 1. 设计定位
定义玩家发布制成品的 WASM 模块与 Profile 治理闭环，统一玩家产物发布、模块约束与身份治理。

## 2. 设计结构
- 玩家产物层：定义玩家发布实体、制成品与其模块/profile 表达。
- 治理审核层：对玩家发布内容执行合法性、权限与版本审核。
- 运行接入层：审核通过后进入运行时模块与实体主路径。
- 审计回滚层：记录发布来源、治理结论与必要回滚信息。

## 3. 关键接口 / 入口
- 玩家发布实体模型
- WASM 模块/Profile 绑定
- 治理审核入口
- 发布审计/回滚记录

## 4. 约束与边界
- 玩家发布内容不得绕过治理审核。
- 实体、模块与 profile 绑定必须可追溯。
- 不在本专题扩展完整 UGC 市场机制。

## 5. 设计演进计划
- 先收敛发布实体模型。
- 再补治理审核与运行接入。
- 最后固定审计和回滚路径。
