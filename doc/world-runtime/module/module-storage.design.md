# 模块存储设计

- 对应需求文档: `doc/world-runtime/module/module-storage.prd.md`
- 对应项目管理文档: `doc/world-runtime/module/module-storage.project.md`

## 1. 设计定位
定义 runtime 模块私有存储、命名空间边界、状态持久化与事件可观测模型。

## 2. 设计结构
- 存储命名空间：按模块隔离状态键空间与访问边界。
- 读写语义：模块通过受控接口读取、更新并持久化状态。
- 观测链路：关键状态变更以事件或审计记录暴露。

## 3. 关键接口 / 入口
- 模块状态读写接口与持久化路径
- `ModuleStateUpdated` 等状态事件
- 模块安装/停用后的状态保留策略

## 4. 约束与边界
- 不允许跨模块直接读写彼此私有存储。
- 状态结构需要兼顾回放、迁移与兼容性。
- 持久化与事件必须可对齐，避免 silent mutation。

- 再按项目文档任务拆解推进实现与验证。
