## 卡片文案

### 第 1 页 / 封面
```text
我现在有这几位 agent 队友
```

补充小字：
```text
一个人，带着一支分工明确的开发队伍做游戏
```

### 第 2 页 / producer_system_designer
```text
负责把目标、规则和玩法口径收在一起

很多看起来是“做不做”的问题，
本质上都是先在这里定方向。
```

### 第 3 页 / runtime_engineer
```text
负责世界怎么运行，系统怎么闭环

很多体验能不能成立，
最后都要回到底层是不是稳。
```

### 第 4 页 / wasm_platform_engineer
```text
负责模块能力和平台边界

它决定的不是表面效果，
而是这套系统以后能不能持续长下去。
```

### 第 5 页 / agent_engineer
```text
负责 agent 的行为设计和能力边界

不是让它“看起来聪明”就够了，
而是让它真的能在世界里行动。
```

### 第 6 页 / viewer_engineer
```text
负责把世界变成可见、可操作、可理解的体验

很多东西做出来以后，
能不能被感受到，要靠这一层完成。
```

### 第 7 页 / qa_engineer
```text
负责验证、拦风险、给出能不能继续往前走的判断

它的价值不只是找错，
而是防止团队对现状产生错觉。
```

### 第 8 页 / liveops_community
```text
负责内容、社区、反馈和对外口径

项目不是做完就结束，
别人怎么理解、怎么回应，也要有人持续接住。
```

### 第 9 页 / 收束页
```text
我负责把他们带着一起往前做

不是单独完成每一件事，
而是让这支分工不同的队伍真的协作起来。
```
