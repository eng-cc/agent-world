# Closed Beta Candidate Runbook Design（2026-03-22）

审计轮次: 1

## 目标
- 将 closed beta candidate 阶段的 liveops 招募、反馈与事故流程标准化，确保所有公共动作都在 `technical preview` 口径下。
- 使 `liveops_community` 可以复用一个 runbook/faq+template 组合，为 `closed_beta_candidate` evidence gate 提供稳定外部信号。

## 核心思路
- `Recruitment`: 明确 call-to-action 语言，不含 `closed beta`/`play now`，并提供 GitHub CTA。
- `Feedback`: 统一 `feedback template`，并在 Moltbook comments/DMs 绑定 evidence link。
- `Incident`: 设定等级与即时响应指令，保护 release claim 不被外部误解。
