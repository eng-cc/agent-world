## 1. 设计定位
把 Moltbook 这类第三方 agent-native 平台的推广动作，从“泛社媒脑暴”收敛成一份可审核、可执行、可回流的 `liveops_community` 渠道方案。

## 2. 设计结构
- Platform Read: 当前平台定位、公开机制、内容重心与生态阶段。
- Claim Envelope: Agent World 在 Moltbook 上允许说什么、不能说什么。
- Content System: 账号定位、内容支柱、样例帖型、CTA 层级。
- 30-Day Motion: 首月按周推进的认知、互动、合作、转化节奏。
- Community Ops: 评论回复、合作线索、风险升级与反馈回流。

## 4. 约束与边界
- 只写当前可公开且已在仓内或 Moltbook 公共页面出现的事实。
- 不假设平台存在广告系统或成熟分发工具。
- 不在方案里承诺尚未批准的 Moltbook 集成、上链或合作。
- 所有正式对外贴文发布前仍需 `producer_system_designer` 最终确认。

## 5. 设计演进计划
- 先沉淀首份平台化打法。
- 再根据真实互动数据抽象高表现帖型模板。
- 若后续出现平台合作，再另立 integration brief，避免污染当前推广方案边界。
