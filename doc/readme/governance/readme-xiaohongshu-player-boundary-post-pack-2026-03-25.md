## 使用说明
- 这是第四篇，主要回答“玩家为什么不能直接控制角色”。
- 目标不是讲输入方案说明书，也不是把玩法系统一次讲完，而是先让用户理解：
  - 玩家在这个世界里不是“操作者”。
  - 玩家能影响什么。
  - 为什么我更想做这种关系，而不是传统的一人一角色控制。
- 语气仍然要像真人开发者在解释自己的设计判断，不像功能文档或产品 FAQ。
- 不要出现以下表述：
  - `已上线`
  - `现在能玩`
  - `公测`
  - `正式发布`
  - `你现在就可以直接体验这套玩法`
- 这一篇只留一个互动问题，不要同时追问“像什么类型”“为什么是小行星带”“什么时候能玩”。

## 正文建议
```text
看完上一条，
你可能也有这个疑问：
玩家到底怎么进这个世界？

答案是：
你不是去附身一个角色。

我不想让玩家
一直盯着某个人，
点移动、点采集、点执行。

我更想做的，
是另一种关系。
你站在世界外面，
给方向，
给目标，
然后看它自己怎么往下长。

所以这里，
玩家不能直接替某个角色行动。

但你也不是纯旁观。
你可以定目标，
给提示，
改走向，
看这个世界最后会变成什么样。

说到底，
我更想让玩家去影响一个世界，
而不是手操一个人。

对我来说，
重点不是
“你按得快不快”，
而是你推的那一下，
会不会真把世界带偏。

如果是你，
你会更想直接控一个角色，
还是在外面给方向？
```
