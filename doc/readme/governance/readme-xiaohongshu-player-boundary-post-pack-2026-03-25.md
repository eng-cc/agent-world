## 使用说明
- 这是第四篇，主要回答“玩家为什么不能直接控制角色”。
- 目标不是讲输入方案说明书，也不是把玩法系统一次讲完，而是先让用户理解：
  - 玩家在这个世界里不是“操作者”。
  - 玩家能影响什么。
  - 为什么我更想做这种关系，而不是传统的一人一角色控制。
- 语气仍然要像真人开发者在解释自己的设计判断，不像功能文档或产品 FAQ。
- 不要出现以下表述：
  - `已上线`
  - `现在能玩`
  - `公测`
  - `正式发布`
  - `你现在就可以直接体验这套玩法`
- 这一篇只留一个互动问题，不要同时追问“像什么类型”“为什么是小行星带”“什么时候能玩”。

## 正文建议
```text
前一条讲了这个游戏大概是什么。

这条想把另一个很容易被问到的问题讲清楚：
为什么玩家不能直接控制一个角色？

因为我不太想做那种
“玩家附身在一个角色身上”的感觉。

我更想让玩家
站在世界外面给方向，
看这个世界自己怎么回应。

所以玩家不是直接操控某个角色去移动、采集、执行每一个动作。

玩家更像一个战略引导者。
你可以定目标，
可以给提示，
可以影响组织结构，
推动这个世界朝某个方向长。

但你不能直接替它活。

我更想看的，
不是玩家手动摆好的结果，
而是一个世界会不会真的自己长出分工、协作和冲突。

所以这里的参与感，
不是“手把手控制”，
而是“你给出去的方向，真的把这个世界带偏了”。

如果是你，
你会更想直接操控角色，
还是站在世界外面给方向？
```
